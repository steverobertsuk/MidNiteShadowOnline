## Overview

Aura/Tint HealthColor is intended to make combat state more readable by mapping token health thresholds to aura colours, tint overlays, or both.

The goal is to keep damage visibility immediate for GMs while staying flexible enough to support different table preferences, monster token setups, and accessibility needs.

## Release notes

### Planned release

- Threshold-driven colour changes for healthy, wounded, bloodied, and critical states.
- Separate controls for aura-only, tint-only, or combined visual modes.
- Support for tokens linked to bars as well as NPC tokens with manually managed values.
